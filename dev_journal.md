- **Среда**: Local Dev (Windows -> Docker)

## 🏗 Архитектурные решения (ADR)
- **ADR-001 (Изоляция)**: Использование Docker и `docker-compose` для полной изоляции среды выполнения OpenClaw.
- **ADR-002 (AI Routing)**: Использование OpenRouter для оплаты по факту использования (Pay-as-you-go). Разделение задач на "Умную модель" (архитектура, логика) и "Быструю модель" (рутина, парсинг).
- **ADR-003 (Prompts as Code)**: Хранение промптов агентов в отдельных файлах в директории `prompts/`.
- **ADR-004 (Human-in-the-loop)**: Все реальные или "виртуальные" финансовые сделки требуют явного подтверждения (Approve/Reject) от пользователя перед выполнением.
- **ADR-005 (Volume Mapping Optimization)**: Все навыки (`skills`) и конфигурации агентов (`agents`) маппятся через Docker Volumes. Это позволяет изменять логику навыков и промпты на хосте без необходимости `docker cp`.
- **ADR-006 (Fault Tolerance Protocol)**: Внедрен протокол немедленного уведомления о сбоях инструментов. Агенты обязаны докладывать Москве (Architect) о любой ошибке API (401, 404, Timeout) до попытки альтернативных действий.

## Инструкции по исправлению Gateway
Если при старте Gateway все еще ругается на конфиг, нужно будет зайти в запущенный контейнер и выполнить настройку:
1. `docker exec -it openclaw_fin_agent bash`
2. `openclaw config set gateway.mode local`
3. `openclaw config set gateway.auth.mode none`
4. Выйти из контейнера: `exit`
5. Перезапустить шлюз: `docker-compose restart`

## 🛡 Работа над ошибками (Lessons Learned)
- **[2026-03-06] Ошибка "Технической слепоты"**: Агенты (Берлин и Лиссабон) столкнулись с ошибками 401 (Perplexity) и 404 (MOEX), но попытались скрыть их за альтернативными методами, не уведомив Москву. Это нарушает принцип прозрачности и самообучаемости. **Урок:** Любой технический сбой должен быть зафиксирован в бэклоге и доложен Профессору. Исправлено через обновление SOUL.md всех агентов.
- **[2026-03-05] Ошибка контекста моделей**: Изначально я перепутал контекст моделей разработки (Cursor) и моделей рантайма агента (OpenClaw). Введено правило непрерывного обучения: при любом недопонимании добавлять уточняющие правила в `.cursorrules`.
- **[2026-03-06] Ошибка сборки Docker (отсутствие git)**: В процессе установки `npm install -g openclaw` возникла ошибка `spawn git ENOENT`. Пакет `git` добавлен в `Dockerfile`.
- **[2026-03-06] Оптимизация Docker Volumes**: Прямой маппинг папок через тома Docker — лучший способ для "вибекодинга", так как изменения в коде на хосте мгновенно доступны в контейнере (после рестарта OpenClaw).
