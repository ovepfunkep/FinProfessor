## 🏗 Архитектурные решения (ADR)
- **ADR-001 (Изоляция)**: Использование Docker и `docker-compose` для полной изоляции среды выполнения OpenClaw.
- **ADR-002 (AI Routing)**: Использование OpenRouter для оплаты по факту использования (Pay-as-you-go). Разделение задач на "Умную модель" (архитектура, логика) и "Быструю модель" (рутина, парсинг).
- **ADR-003 (Prompts as Code)**: Хранение промптов агентов в отдельных файлах в директории `prompts/`.
- **ADR-004 (Human-in-the-loop)**: Все реальные или "виртуальные" финансовые сделки требуют явного подтверждения (Approve/Reject) от пользователя перед выполнением.

## Инструкции по исправлению Gateway
Если при старте Gateway все еще ругается на конфиг, нужно будет зайти в запущенный контейнер и выполнить настройку:
1. `docker exec -it openclaw_fin_agent bash`
2. `openclaw config set gateway.mode local`
3. `openclaw config set gateway.auth.mode none`
4. Выйти из контейнера: `exit`
5. Перезапустить шлюз: `docker-compose restart`

## 🛡 Работа над ошибками (Lessons Learned)
- **[2026-03-05] Ошибка контекста моделей**: Изначально я перепутал контекст моделей разработки (Cursor) и моделей рантайма агента (OpenClaw). Введено правило непрерывного обучения: при любом недопонимании добавлять уточняющие правила в `.cursorrules`. Для MVP OpenClaw зафиксировано использование сверхдешевых/бесплатных моделей (напр. MiniMax M2.5) через OpenRouter.
- **[2026-03-05] Архитектура OpenClaw**: Скорректирован Dockerfile с Python на Node.js (v22), так как ядро фреймворка OpenClaw написано на TypeScript/Node.js. Создан конфигурационный файл `SOUL.md` (базовый манифест агента).
- **[2026-03-05] User Action Required**: Введен паттерн выноса ручных действий пользователя в отдельный файл `USER_ACTION_REQUIRED.md`. Цель: запуск "Нулевого этапа" для проверки общения с агентом.
- **[2026-03-06] Ошибка фиксации ошибок (Рекурсия)**: Я упомянул о допущенной ошибке в диалоге, но забыл зафиксировать правило в "Работе над ошибками" (обязательная сверка с документацией перед написанием Dockerfile). Введено правило: *каждая* найденная ошибка должна превращаться в правило и запись в дневник без исключений.
- **[2026-03-06] Изменение стиля общения и кода**: По запросу пользователя добавлен стиль общения с теплотой и легким юмором (5%). В код-стайл добавлено требование писать исчерпывающие комментарии и подробное логирование, чтобы любой новый агент мог моментально вникнуть в процесс.
- **[2026-03-06] Интеграция Telegram**: Добавлен канал связи через Telegram в `SOUL.md` и соответствующая переменная окружения в `.env`. Теперь агент может общаться не только через веб-интерфейс, но и через мессенджер.
- **[2026-03-06] Ошибка сборки Docker (отсутствие git)**: В процессе установки `npm install -g openclaw` возникла ошибка `spawn git ENOENT`. Это связано с тем, что пакет OpenClaw тянет зависимости через git. Добавлено правило: всегда включать `git` в базовый образ Docker при установке npm-пакетов в среде `node-slim`. Пакет `git` добавлен в `Dockerfile`.
- **[2026-03-06] Регистрация устройства и модели агента**: По умолчанию OpenClaw пытается использовать `anthropic/claude-opus-4-6`. Для использования бесплатной модели Minimax через OpenRouter была добавлена команда `openclaw models set openrouter/minimax/minimax-m2.5` (по явному требованию пользователя) и регистрация токена OpenRouter через `openclaw models auth paste-token --provider openrouter`. Модель успешно переключена.
